# ScraperBackendService[中文](#chinese) | [English](#english)---<a name="chinese"></a>## 中文HanimetaScraper 后端服务，提供 Hanime 和 DLsite 元数据 REST API。### 功能- 多提供商支持 - Hanime、DLsite 统一接口- RESTful API - 标准化 JSON 响应- 反机器人 - Playwright 隐身模式- 智能缓存 - LRU 策略- 并发控制 - 全局并发限制- 速率限制 - 每槽独立速率控制- 身份验证 - 可选 API Token### API 端点**基础：**- `GET /` - 服务信息- `GET /health` - 健康检查- `GET /cache/stats` - 缓存统计- `DELETE /cache/clear` - 清空缓存**Hanime：**- `GET /api/hanime/search?title={query}&max={limit}`- `GET /api/hanime/{id}`**DLsite：**- `GET /api/dlsite/search?title={query}&max={limit}`- `GET /api/dlsite/{id}`- `GET /r/dlsite/{id}` - 重定向### 配置编辑 `appsettings.json`：```json{  "ServiceConfig": {    "Port": 8585,    "Host": "0.0.0.0",    "AuthToken": "",    "MaxConcurrentRequests": 3,    "RateLimitSeconds": 20,    "RequestTimeoutSeconds": 180  }}```**配置说明：**| 配置项 | 描述 | 默认值 | 推荐范围 ||--------|------|--------|----------|| `Port` | HTTP 监听端口 | 8585 | 1024-65535 || `Host` | 监听地址 | "0.0.0.0" | 127.0.0.1（本地）<br>0.0.0.0（所有接口） || `AuthToken` | API 认证令牌 | "" | **强随机字符串**<br>（生产环境必须设置） || `MaxConcurrentRequests` | 全局并发槽位数 | 3 | 1-10 || `RateLimitSeconds` | 全局速率限制（秒）<br>每槽最小请求间隔 | 20 | 10-60<br>0=禁用 || `RequestTimeoutSeconds` | 请求超时（秒） | 180 | 90-300 || `EnableAggressiveMemoryOptimization` | 激进内存优化<br>频繁 GC | true | true/false || `ChallengeAutoWaitSeconds` | Cloudflare 挑战等待（秒）<br>首次尝试 | 20 | 10-30 || `ChallengeAutoWaitSlowSeconds` | Cloudflare 挑战等待（秒）<br>慢速重试 | 45 | 30-60 |> **重要**：`MaxConcurrentRequests` 和 `RateLimitSeconds` 对所有提供商（Hanime、DLsite）统一生效。**并发与速率限制：**- **并发控制**：限制同时访问的请求数  - 达到限制时等待最多 15 秒  - 超时返回 429 状态码  - **速率限制**：每个槽的最小请求间隔  - 缓存命中时绕过限制  - 设置为 0 禁用  - 每槽独立计时**性能调优：**```json// 个人使用{ "MaxConcurrentRequests": 3, "RateLimitSeconds": 20 }// 多用户{ "MaxConcurrentRequests": 10, "RateLimitSeconds": 20 }// 保守（避免封禁）{ "MaxConcurrentRequests": 1, "RateLimitSeconds": 60 }```**环境变量：**| 变量 | 对应配置 ||------|----------|| `SCRAPER_PORT` | Port || `SCRAPER_AUTH_TOKEN` | AuthToken |### 部署```bash# 开发dotnet run# 生产dotnet publish -c Release -o ./publishcd publish && dotnet ScraperBackendService.dll```### 监控```bash# 健康检查curl http://localhost:8585/health# 缓存统计curl http://localhost:8585/cache/stats# 清空缓存curl -X DELETE http://localhost:8585/cache/clear```### 故障排除| 问题 | 解决方案 ||------|----------|| 503 Service Unavailable | 增加 `MaxConcurrentRequests` || 请求超时 | 增加 `RequestTimeoutSeconds`<br>检查网络 || 频繁 429 错误 | 减少 `RateLimitSeconds`<br>增加 `MaxConcurrentRequests` || 内存占用高 | 启用 `EnableAggressiveMemoryOptimization` |**日志级别：**```json{  "Logging": {    "LogLevel": {      "Default": "Information",      "ScraperBackendService": "Debug"    }  }}```### 安全**生产部署：**1. 设置强 `AuthToken`2. 防火墙限制访问3. 使用 HTTPS（Nginx 反向代理）4. 监控日志**Nginx 示例：**```nginxserver {    listen 443 ssl;    server_name your-domain.com;    ssl_certificate /path/to/cert.pem;    ssl_certificate_key /path/to/key.pem;        location / {        proxy_pass http://127.0.0.1:8585;        proxy_set_header Host $host;        proxy_set_header X-Real-IP $remote_addr;    }}```### 技术栈- .NET 9 - Hanime, DLsite unified interface- RESTful API - Standardized JSON responses- Anti-bot - Playwright stealth mode- Smart caching - LRU strategy- Concurrency control - Global limit- Rate limiting - Per-slot rate control- Authentication - Optional API Token### API Endpoints**Base:**- `GET /` - Service info- `GET /health` - Health check- `GET /cache/stats` - Cache stats- `DELETE /cache/clear` - Clear cache**Hanime:**- `GET /api/hanime/search?title={query}&max={limit}`- `GET /api/hanime/{id}`**DLsite:**- `GET /api/dlsite/search?title={query}&max={limit}`- `GET /api/dlsite/{id}`- `GET /r/dlsite/{id}` - Redirect### ConfigurationEdit `appsettings.json`:```json{  "ServiceConfig": {    "Port": 8585,    "Host": "0.0.0.0",    "AuthToken": "",    "MaxConcurrentRequests": 3,    "RateLimitSeconds": 20,    "RequestTimeoutSeconds": 180  }}```**Configuration Options:**| Setting | Description | Default | Recommended ||---------|-------------|---------|-------------|| `Port` | HTTP listening port | 8585 | 1024-65535 || `Host` | Listening address | "0.0.0.0" | 127.0.0.1 (local)<br>0.0.0.0 (all interfaces) || `AuthToken` | API auth token | "" | **Strong random string**<br>(Required for production) || `MaxConcurrentRequests` | Global concurrency slots | 3 | 1-10 || `RateLimitSeconds` | Global rate limit (seconds)<br>Min interval per slot | 20 | 10-60<br>0=disabled || `RequestTimeoutSeconds` | Request timeout (seconds) | 180 | 90-300 || `EnableAggressiveMemoryOptimization` | Aggressive memory optimization<br>Frequent GC | true | true/false || `ChallengeAutoWaitSeconds` | Cloudflare challenge wait (sec)<br>First attempt | 20 | 10-30 || `ChallengeAutoWaitSlowSeconds` | Cloudflare challenge wait (sec)<br>Slow retry | 45 | 30-60 |> **Important**: `MaxConcurrentRequests` and `RateLimitSeconds` apply globally to all providers (Hanime, DLsite).**Concurrency & Rate Limiting:**- **Concurrency Control**: Limits simultaneous requests  - Waits up to 15s when limit reached  - Returns 429 on timeout  - **Rate Limiting**: Min interval per slot  - Bypassed on cache hit  - Set to 0 to disable  - Independent timer per slot**Performance Tuning:**```json// Personal use{ "MaxConcurrentRequests": 3, "RateLimitSeconds": 20 }// Multi-user{ "MaxConcurrentRequests": 10, "RateLimitSeconds": 20 }// Conservative (avoid blocking){ "MaxConcurrentRequests": 1, "RateLimitSeconds": 60 }```**Environment Variables:**| Variable | Config ||----------|--------|| `SCRAPER_PORT` | Port || `SCRAPER_AUTH_TOKEN` | AuthToken |### Deployment```bash# Developmentdotnet run# Productiondotnet publish -c Release -o ./publishcd publish && dotnet ScraperBackendService.dll```### Monitoring```bash# Health checkcurl http://localhost:8585/health# Cache statscurl http://localhost:8585/cache/stats# Clear cachecurl -X DELETE http://localhost:8585/cache/clear```### Troubleshooting| Issue | Solution ||-------|----------|| 503 Service Unavailable | Increase `MaxConcurrentRequests` || Request timeouts | Increase `RequestTimeoutSeconds`<br>Check network || Frequent 429 errors | Decrease `RateLimitSeconds`<br>Increase `MaxConcurrentRequests` || High memory usage | Enable `EnableAggressiveMemoryOptimization` |**Log Levels:**```json{  "Logging": {    "LogLevel": {      "Default": "Information",      "ScraperBackendService": "Debug"    }  }}```### Security**Production Deployment:**1.
