# ScraperBackendService[中文](#中文) | [English](#english)---## 中文`ScraperBackendService` 是 HanimetaScraper 的后端服务，提供 Hanime 和 DLsite 的统一抓取 API。### 运行环境- .NET 9- Playwright Chromium（用于动态页面抓取）### 启动```bashcd ScraperBackendServicedotnet run```默认地址：`http://0.0.0.0:8585`也可通过环境变量覆盖：- `SCRAPER_PORT`- `SCRAPER_AUTH_TOKEN`### API 列表#### 服务接口- `GET /`  - 返回服务信息（含 `authEnabled`）- `GET /health`  - 健康检查#### 缓存接口- `GET /cache/stats`  - 缓存命中/未命中/淘汰统计- `DELETE /cache/clear`  - 清空缓存- `DELETE /cache/{provider}/{id}`  - 删除单条缓存#### Hanime- `GET /api/hanime/search?title={query}&max={limit}`- `GET /api/hanime/{id}`#### DLsite- `GET /api/dlsite/search?title={query}&max={limit}`- `GET /api/dlsite/{id}`- `GET /r/dlsite/{id}`（302 重定向到 DLsite 详情页）### 认证规则当 `ServiceConfig.AuthToken` 为空时：不启用认证。当 `ServiceConfig.AuthToken` 非空时：- `/api/*` 与 `/cache/*` 必须携带请求头：`TokenHeaderName: <token>`- `/`、`/health`、`/r/dlsite/*` 不受 Token 认证拦截示例（默认头名）：```bashcurl -H "X-API-Token: your-token" "http://127.0.0.1:8585/api/hanime/86994"```### 配置（`appsettings.json`）```json{  "ServiceConfig": {    "Port": 8585,    "Host": "0.0.0.0",    "AuthToken": "",    "TokenHeaderName": "X-API-Token",    "MaxConcurrentRequests": 2,    "RateLimitSeconds": 5,    "RequestTimeoutSeconds": 180,    "EnableAggressiveMemoryOptimization": true,    "ChallengeAutoWaitSeconds": 30,    "ChallengeAutoWaitSlowSeconds": 60,    "EnableManualChallengeResolution": true,    "ManualChallengeTimeoutSeconds": 180,    "ManualChallengeWindowWidth": 1280,    "ManualChallengeWindowHeight": 900,    "EnableCookiePersistence": true,    "AutoLoadCookiesOnContextCreation": true,    "PreferSystemChromeForHeadless": true,    "CacheTtlMinutes": 2,    "CacheNullResultTtlMinutes": 2,    "CacheSizeLimitEntries": 100  }}```### 常见响应状态- `200`：成功- `400`：请求参数非法（如 `title` 为空、ID 无效）- `401`：缺失或错误 Token- `404`：内容不存在- `429`：并发槽位繁忙- `500`：内部错误- `504`：请求超时（由全局超时中间件返回）---## English`ScraperBackendService` is the backend service of HanimetaScraper, exposing unified scraping APIs for Hanime and DLsite.### Runtime- .NET 9- Playwright Chromium (for dynamic page scraping)### Start```bashcd ScraperBackendServicedotnet run```Default URL: `http://0.0.0.0:8585`You can override via environment variables:- `SCRAPER_PORT`- `SCRAPER_AUTH_TOKEN`### API List#### Service Endpoints- `GET /`  - Service info (includes `authEnabled`)- `GET /health`  - Health check#### Cache Endpoints- `GET /cache/stats`  - Cache hit/miss/eviction statistics- `DELETE /cache/clear`  - Clear all cache entries- `DELETE /cache/{provider}/{id}`  - Remove a single cache entry#### Hanime- `GET /api/hanime/search?title={query}&max={limit}`- `GET /api/hanime/{id}`#### DLsite- `GET /api/dlsite/search?title={query}&max={limit}`- `GET /api/dlsite/{id}`- `GET /r/dlsite/{id}` (302 redirect to DLsite detail page)### Authentication RulesWhen `ServiceConfig.AuthToken` is empty: authentication is disabled.When `ServiceConfig.AuthToken` is set:- `/api/*` and `/cache/*` require header: `TokenHeaderName: <token>`- `/`, `/health`, and `/r/dlsite/*` are not blocked by token middlewareExample (default header name):```bashcurl -H "X-API-Token: your-token" "http://127.0.0.1:8585/api/hanime/86994"```### Configuration (`appsettings.json`)```json{  "ServiceConfig": {    "Port": 8585,    "Host": "0.0.0.0",    "AuthToken": "",    "TokenHeaderName": "X-API-Token",    "MaxConcurrentRequests": 2,    "RateLimitSeconds": 5,    "RequestTimeoutSeconds": 180,    "EnableAggressiveMemoryOptimization": true,    "ChallengeAutoWaitSeconds": 30,    "ChallengeAutoWaitSlowSeconds": 60,    "EnableManualChallengeResolution": true,    "ManualChallengeTimeoutSeconds": 180,    "ManualChallengeWindowWidth": 1280,    "ManualChallengeWindowHeight": 900,    "EnableCookiePersistence": true,    "AutoLoadCookiesOnContextCreation": true,    "PreferSystemChromeForHeadless": true,    "CacheTtlMinutes": 2,    "CacheNullResultTtlMinutes": 2,    "CacheSizeLimitEntries": 100  }}```### Common Status Codes- `200`: success- `400`: invalid request (e.g., empty `title`, invalid ID)- `401`: missing or invalid token- `404`: content not found- `429`: all concurrency slots busy- `500`: internal error- `504`: request timeout (from global timeout middleware)
